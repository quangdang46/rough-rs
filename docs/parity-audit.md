# rough.js parity audit

rough-rs uses `legacy/rough` rough.js 4.6.6 as its executable reference.
The committed fixture matrix in `tests/fixtures/reference.json` is generated by
`verify/generate_reference.mjs`.

## Comparison contract

The strict parity gate is `cargo test --test comprehensive_parity`.
It renders every committed reference fixture through the public Rust API and
checks:

- shape names and op-set ordering
- op-set types and op counts
- op types and numeric op data with `1e-10` tolerance
- finite SVG path data and SVG attribute sanity
- seeded determinism across representative option matrices
- an embeddable SVG sample containing multiple primitives

The tolerance is intentionally numeric rather than byte-for-byte SVG string
identity. Rust and JavaScript format floating point path strings differently in
some cases, while the underlying op data can still match the rough.js reference.

## Accepted divergence

`rectangle_fill_dots` is compared by structure and op count, but not by exact dot
coordinates. In rough.js 4.6.6, `DotFiller` places dot centers with
`Math.random()` instead of the seeded rough.js randomizer. rough-rs uses its
per-shape `RngHelper` so seeded dot fills are deterministic. Downstream
Excalidraw-style use benefits from this determinism, and the rendered structure,
op-set type, and op count still match the reference fixture.

No other accepted divergence is currently recorded.
